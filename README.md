# Sunstead

Firecrawl-inspired landing page and component-library study built with TanStack Start,
Vite+, React, Tailwind CSS, and Storybook.

The current UI direction uses:

- pale canvas background with thin neutral borders
- heat-orange accent for primary actions and emphasized words
- compact sticky navigation and announcement bar
- centered first-viewport hero with a grid backdrop
- browser/code mockups, product cards, solution tables, FAQ, and footer primitives
- Storybook stories for the reusable marketing components

## Stack

- TanStack Start + TanStack Router
- Vite+ command surface over Vite, Vitest, Oxlint, and Oxfmt
- Bun for package management and scripts
- Tailwind CSS v4
- Storybook with the React/Vite builder

## Dev Workflow

Use DevMe as the only way to run local servers in this repo. It owns the
processes, assigns the ports, and keeps logs in one place.

Start or repair the environment:

```bash
devme up -d
devme status
```

Open the running services from DevMe, never by guessing a port:

```bash
devme url web
devme url storybook
```

Read logs from DevMe:

```bash
devme logs web --since 5m
devme logs storybook --since 5m
devme logs --since 5m
devme doctor
```

If a service is stale or broken, restart it through DevMe:

```bash
devme restart web
devme restart storybook
```

Do not run `vp dev`, `bun run dev`, `vp run storybook`, or `storybook dev`
directly. Those commands create unmanaged servers on fixed ports like `3000` or
`6006`, which makes agents and browsers disagree about which app is current.
If you find a manual server already running, stop that process and bring the
stack back with `devme up -d`.

Use Vite+ for one-shot maintenance commands only:

```bash
vp install
vp check
vp test
vp build
vp run build-storybook
```

Port rule: each worktree gets its own DevMe slot. The main app might be on
`3040` here and a sibling worktree might be on another port. That is expected.
The source of truth is always `devme status` or `devme url <service>`.
