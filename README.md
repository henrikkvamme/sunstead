# Sunstead

Example TanStack Start app built with Vite+, Bun, Tailwind CSS, Base UI, Drizzle,
and SQLite/libSQL.

The home route is a small plant-care console that demonstrates:

- A TanStack Router route loader calling a TanStack Start `GET` server function.
- `POST` server functions for creating, updating, and deleting database rows.
- Zod validation at the server-function boundary.
- Drizzle ORM talking to SQLite/libSQL through `@libsql/client`.
- Base UI primitives (`Field`, `Input`, `Switch`, and `Tabs`) styled with Tailwind CSS v4.

## Stack

- TanStack Start + TanStack Router
- Vite+ command surface over Vite, Vitest, Oxlint, and Oxfmt
- Bun for package management and scripts
- Tailwind CSS v4 with Base UI primitives for custom-styled accessible UI
- Zod for server/env/request validation
- Better Auth dependency and env placeholders
- Drizzle ORM with SQLite/libSQL

## Example Flow

The database example lives in `src/lib/plants.ts`.

- `listPlants` seeds demo rows if the table is empty, then returns serialized rows.
- `createPlant` validates form input with Zod and inserts a row.
- `waterPlant` and `flagPlantForWater` update boolean/timestamp fields.
- `deletePlant` removes a row by id.

The UI example lives in `src/routes/index.tsx`.

- The route loader reads from `listPlants`.
- Form submit calls `createPlant`, then invalidates the router.
- Switch changes call `waterPlant` or `flagPlantForWater`.
- Delete buttons call `deletePlant`.

## Commands

```bash
./node_modules/.bin/vp install
bun run dev
bun run check
bun run test
bun run build
```

Database commands:

```bash
bun run db:generate
bun run db:migrate
bun run db:studio
```

## Development Environment

This repo includes `devme.toml`.

```bash
devme up -d
devme url web -o
devme doctor
```

On first run, DevMe writes missing local values to `.env.local`, including
`BETTER_AUTH_SECRET`.

## Project Notes

- Oxlint and Oxfmt are configured in `vite.config.ts`, following Vite+ convention.
- `src/routeTree.gen.ts` is generated by TanStack Router and excluded from formatter checks.
- TanStack Start/Router packages are pinned because Start is still moving quickly.
- The default database URL is `file:./local.db`; switch it to a remote libSQL/Turso URL when
  deploying.
