# Project Skills

This repo vendors the workflow skills that are expected to be useful for Sunstead work.

## Product and issue workflow

- `grill-with-docs` - sharpen the domain language and update `CONTEXT.md`/ADRs as decisions settle.
- `to-prd` - turn a product plan into a PRD in the configured issue tracker.
- `to-issues` - split a PRD or plan into Linear issues.
- `triage` - move incoming work through the canonical triage states in `docs/agents/triage-labels.md`.
- `setup-matt-pocock-skills` - refresh `AGENTS.md` and `docs/agents/` if the tracker or domain-doc layout changes.

## Engineering workflow

- `tdd` - build features or fixes with a red-green-refactor loop.
- `diagnose` - debug broken behavior with a reproducible feedback loop.
- `improve-codebase-architecture` - find deeper refactoring opportunities after the domain model is clearer.
- `zoom-out` - explain an unfamiliar code area in broader context.

## Shared helpers

- `domain-modeling` - shared glossary and ADR formats.
- `codebase-design` - shared deep-module vocabulary.
- `devme` - runtime, service, log, restart, and environment debugging workflow.
